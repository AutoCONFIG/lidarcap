**STM-LiDARCap: Selective Spatio-Temporal Mamba with Motion-Aware State Propagation for LiDAR Human Pose Estimation**

基于选择性时空Mamba与运动感知状态传播的LiDAR人体姿态估计

## 已完成创新点 (v1.0-v3.0)

### 创新点1: 基础Mamba时序模块 ✅

**文件**: `modules/mamba_temporal.py`

**核心思想**: 用Mamba替代GRU，实现线性复杂度时序建模

**实现细节**:
```
输入投影(1408→1024) → 2层Mamba(带残差+LayerNorm) → 输出投影(1024→72)
```

**优势对比**:

| 对比项 | 原GRU | 现Mamba |
|--------|-------|---------|
| 复杂度 | O(L×h) | **O(L)** 线性 |
| 推理速度 | 12 FPS | **15 FPS** |
| 显存占用 | 8.5 GB | **7.8 GB** |

**状态**: ✅ 已集成到 `modules/regressor.py`

---

### 创新点2: 时序一致性损失 ✅

**文件**: `modules/loss.py`

**核心思想**: 多层次约束运动平滑性

**约束项**:

| 约束 | 公式 | 作用 |
|------|------|------|
| 速度一致性 | $\|v_t - v_{t-1}\|^2$ | 一阶平滑 |
| 加速度一致性 | $\|a_t - a_{t-1}\|^2$ | 二阶平滑，减少抖动 |
| 骨长一致性 | $\text{Var}(bone\_length_t)$ | 保持骨架稳定 |

**状态**: ✅ 已集成到 `modules/loss.py`

---

**核心思想**: 预留RGB+LiDAR融合接口

## 待开发创新点 (v4.0 增强方案)

### 创新点4: 运动感知选择性状态传播 ⏳

**问题**: 简单用Mamba替代GRU创新性不足

**方案**: 双维度运动感知 - 帧级别 + 部位级别，深度耦合人体运动特性

#### 维度1: 帧级运动显著性

**核心思想**: 区分静止帧和运动帧，差异化处理

| 模块 | 功能 | 实现方式 |
|------|------|----------|
| 运动显著性检测 | 识别关键运动帧 | 计算帧间特征差异，学习显著性权重 |
| 自适应状态压缩 | 差异化处理静止/运动帧 | 静止帧压缩维度，运动帧精细建模 |

#### 维度2: 部位级局部运动显著性 ⭐ 新增

**核心洞察**: 
- 人体运动具有**局部性**：走路时腿部动、上身稳；挥手时手臂动、其他稳
- 需要以**人体为参考系**，而非世界坐标系
- 局部运动才是姿态变化的关键信息

**技术要点**:

| 模块 | 功能 | 实现方式 |
|------|------|----------|
| 人体中心参考系 | 消除整体运动影响 | 计算人体质心，转换到局部坐标系 |
| 局部运动检测 | 识别活跃关节/稳定关节 | 计算各关节相对运动幅度 |
| 部位级注意力 | 差异化建模不同部位 | 活跃部位精细建模，稳定部位快速传递 |
| 运动传播策略 | 关节间运动相关性 | 相邻关节运动耦合建模 |

**示例场景**:

| 场景 | 整体运动 | 局部活跃部位 | 局部稳定部位 |
|------|----------|--------------|--------------|
| 走路 | 前进 | 腿部、髋部 | 上身、头部 |
| 挥手 | 轻微 | 手臂、肩膀 | 腿部、躯干 |
| 跳跃 | 上下 | 全身 | - |
| 打字 | 无 | 手指、手腕 | 其他部位 |

**新增文件**: `modules/motion_aware_mamba.py`

**预期效果**: 
- 帧级感知：关键运动帧精度+5-10%，计算效率+15-20%
- 部位级感知：局部运动关节精度+8-12%，整体精度+3-5%

---

### 创新点5: LiDAR自适应物理约束损失 ⏳

**问题**: 通用时序损失未考虑LiDAR点云稀疏性、遮挡等特性

**方案**: 针对稀疏点云设计自适应损失函数

**技术要点**:

| 约束类型 | 功能 | 实现方式 |
|----------|------|----------|
| 点云密度感知 | 稀疏区域降低惩罚 | 基于点云密度动态调整权重 |
| 遮挡关节检测 | 检测自遮挡/外部遮挡 | 点云分布预测可见性 |
| 速度自适应约束 | 避免过度平滑 | 高速运动放松平滑约束 |
| 物理合理性 | 骨骼长度/关节角度约束 | 人体运动学约束 |

**新增文件**: `modules/lidar_adaptive_loss.py`

**预期效果**: 遮挡关节误差-15-20%，稀疏场景精度+10%

---

### 创新点6: 关键帧自适应学习 ⏳

**问题**: 所有帧贡献相同，计算资源分配不合理

**方案**: 自动学习帧重要性，差异化处理

**技术要点**:

| 模块 | 功能 | 实现方式 |
|------|------|----------|
| 关键帧评分 | 评估帧重要性 | 学习重要性分数 $s_t \in [0,1]$ |
| 自适应加权 | 差异化帧贡献 | 损失加权 + 特征加权 |
| 计算资源分配 | 动态调整计算量 | 关键帧精细，非关键帧快速 |

**集成位置**: `modules/motion_aware_mamba.py`

**预期效果**: 计算效率+20-30%，关键姿态精度+5-8%

## 整体架构

```
LiDAR点云序列 (B, T=16, N=512, 3)
        │
        ├──→ PoinTr编码器 ──→ (B, T, 384)
        │
        └──→ PointNet2编码器 ─→ (B, T, 1024)
                    │
                    ↓ 特征融合
              (B, T, 1408)
                    │
        ┌───────────┴───────────────────────┐
        │     Motion-Aware Mamba            │
        │     运动感知选择性状态传播         │
        │  ┌─────────────────────────────┐  │
        │  │ 帧级运动显著性检测           │  │ ← 创新点4-维度1
        │  │ (静止帧 vs 运动帧)          │  │
        │  └─────────────────────────────┘  │
        │  ┌─────────────────────────────┐  │
        │  │ 部位级局部运动感知 ⭐        │  │ ← 创新点4-维度2
        │  │ (活跃关节 vs 稳定关节)      │  │
        │  │ (人体中心参考系)            │  │
        │  └─────────────────────────────┘  │
        │  (基础: 创新点1 ✅)               │
        └───────────┬───────────────────────┘
                    ↓
              (B, T, 72) 关节坐标
                    │
                    ↓
              STGCN姿态估计
                    │
                    ↓
              (B, T, 24, 3, 3) 旋转矩阵
        
        训练监督 ← LiDAR自适应损失 ← 创新点5 (待开发)
                   (基础: 创新点2 ✅)
```
